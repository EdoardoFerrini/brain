State Management

Apr 02, 20261 min read

Gestire e sincronizzare i dati nell’applicazione

Tipi di stato

stato locale — appartiene a un componente, stato condiviso — condiviso tra più componenti, stato server — dati che vengono dal backend, stato URL — dati nella query string e nei path params

Stato locale

useState per stato semplice, useReducer per stato complesso, quando alzare lo stato (lifting state up)

Context API

quando usare il context, createContext e Provider, performance — context e re-render, splitting dei context per dominio

Stato globale client

Zustand — store minimale e flessibile, Jotai — stato atomico, Redux Toolkit — per app complesse, quando serve davvero uno stato globale

Stato server

cos’è il server state (dati remoti, asincroni, cache), React Query — fetching, caching, invalidation, SWR — stale-while-revalidate, ottimistic updates, sincronizzazione e refetch automatico

Stato URL

query params come stato, useSearchParams in React Router, quando preferire l’URL allo stato in memoria
