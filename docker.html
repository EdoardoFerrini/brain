Docker

Apr 02, 20263 min read

Containerizzare applicazioni per ambienti riproducibili e portabili

Concetti fondamentali

container — processo isolato con il proprio filesystem immagine — template immutabile per creare container dockerfile — istruzioni per costruire un’immagine registry — repository di immagini (Docker Hub, GitHub Container Registry) differenza tra VM e container — container condivide il kernel dell’host

Dockerfile

FROM — immagine base WORKDIR — directory di lavoro COPY e ADD — copiare file nell’immagine RUN — eseguire comandi durante la build ENV — variabili d’ambiente EXPOSE — porta che il container ascolta (documentazione) CMD vs ENTRYPOINT — differenza e combinazione .dockerignore — escludere file dalla build

Layer e Cache

ogni istruzione crea un layer i layer sono cachati — cambiare un layer invalida tutti quelli successivi ordinare le istruzioni per sfruttare la cache (dipendenze prima del codice) layer grandi — copiare node_modules vs installare dentro il container

Immagini

docker build -t nome:tag — costruire un’immagine docker images — listare le immagini locali docker push — scaricare e caricare dal registry tag latest — convenzione, non garantisce che sia la versione più recente image digest — hash immutabile dell’immagine

Container

docker run — creare e avviare un container docker run -d — detached mode (background) docker run -p 8080:3000 — port mapping host:container docker run -v — mount di volumi docker run —env-file — caricare variabili d’ambiente docker ps — container in esecuzione docker exec -it — entrare in un container in esecuzione docker logs — vedere i log del container docker rm — fermare e rimuovere

Storage

volumi — gestiti da Docker, persistenti tra container bind mount — montare una cartella dell’host nel container tmpfs mount — solo in memoria, non persistente quando usare volumi vs bind mount

Networking

bridge network — default, container comunicano per nome host network — container usa direttamente la rete dell’host port mapping — esporre porte del container sull’host DNS interno Docker — container si trovano per nome del servizio

Docker Compose

orchestrare più container con un file YAML services, networks, volumes logs depends_on — ordine di avvio dei servizi environment e env_file override con docker-compose.override.yml

Multi-stage Build

usare più FROM in un Dockerfile build stage — compilare l’applicazione runtime stage — copiare solo il necessario immagini finali molto più leggere esempio: build con Node → runtime con nginx

Produzione

immagini minimali — alpine, distroless non eseguire come root — USER instruction healthcheck — verificare che il container sia sano secrets — non mettere credenziali nell’immagine stderr — lasciare che Docker gestisca i log
